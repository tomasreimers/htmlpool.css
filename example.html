HTMLPool Dive into HTML.

So first things first, welcome to the HTP Demo!

In every web language, developers have worked to make amazing boilerplates and defaults:

  • In Javascript, frameworks like jQuery and Prototype have added key functionality such as better selectors and brief ajax;
  • In HTML, boilerplates like Initializr, HTML5Boilerplate, and HTMLShell add key default code;
  • And in CSS, we have resets and frameworks (i.e. Bootstrap or jQuery Mobile)
But what about a good set of CSS Defaults? Most designers and developers wouldn't be caught dead using the default serifed font or the default table and input elements; nevertheless, there seems to be no boilerplate of good style defaults unless you want to use an entire application framework (i.e. Bootstrap or jQuery Mobile).

Enter HTMLPool.

HTMLPool helps you dive into HTML by providing a set of styling defaults you wouldn't be ashamed to use, without bringing the bulkiness of a framework.

Additionally, this entire demo has no custom CSS! It is pure HTML. Don't believe me? Look at the source...

Usage

Simply include the single HTMLPool stylesheet, as such: 

<link href='htmlpool.css' rel='stylesheet' type='text/css' />
Watch as your prototypes and unstyled pages transform into pretty pages; then add and override style as necessary. HTMLPool was written with the intention to be overwritten, so all of its properties are defined as loosely as possible to allow you to override them with even the weakest of selectors. Lastly, HTMLPool was designed to be lightweight, so it shouldn't affect your page load.

Horizontal Rule:

Common Classes

These classes are not classes you might use, they are classes I predict you will use; unless I expect at least 50% of HTMLPool users will use a class, I won't add it. (Remember this is a boilerplate, not a framework.)

  • clearfix: Your standard clearfix.
  • wrapper: A standard margin: auto wrapper with a max width of 960px.
